# Map Detection

Detects the map in raw board game photos using a YOLOv8 object detection model, producing a bounding box around the map in each image.

## What it does

Given a photo of a board game in play, this module outputs an axis-aligned bounding box identifying the map's location. The detector is a fine-tuned YOLOv8n model trained on images of the map from various angles and lighting conditions.

## Where it fits in the pipeline

- **Input**: Raw game photos (JPG/PNG)
- **Output**: Bounding box coordinates `[x1, y1, x2, y2]` for the map region
- **Feeds into**: Downstream modules that need to isolate and read the map (e.g. territory/resource state extraction)

## How to run

### 1. Augment training data (optional)

```bash
cd map_detection/src
python data_augmentation.py
```

Reads images and YOLO labels from `AUG_DATASET_DIR` (set in `.env`), applies random flips and rotations, and writes augmented copies to `images_aug/` and `labels_aug/` inside the same directory.

Tune augmentation behaviour directly in `data_augmentation.py`:
- `N_AUGS` ‚Äî number of augmented copies per image (default `10`)

### 2. Train the model

Open and run `src/map_detection.ipynb` cell by cell:

| Step | What it does |
|------|-------------|
| 1 | Resolves dataset paths from `.env` |
| 2 | Splits data 80/20 into train/val |
| 3 | Writes `data.yaml` for YOLO |
| 4 | Auto-detects GPU / falls back to CPU |
| 5 | Trains YOLOv8 ‚Äî tune `EPOCHS`, `IMG_SIZE`, `BATCH_SIZE`, `DEVICE`, `RUN_NAME` here |
| 6 | Validates and prints mAP metrics |
| 7 | Runs sample inference |
| 8 | Copies `best.pt` ‚Üí `MODEL_SAVE_DIR/MODEL_SAVE_NAME` |
| 9-10 | Evaluates per-image IoU and visualises bad detections |

### 3. Run inference with a trained model

```python
from ultralytics import YOLO

model = YOLO("saved_models/map_detector_v1.pt")
results = model("path/to/image.jpg")
boxes = results[0].boxes  # [x1, y1, x2, y2] bounding boxes
```

## Setup

### Environment

```bash
conda create -n futurescape python=3.9
conda activate futurescape
pip install -r ../requirements.txt
```

### Configure paths

Edit `map_detection/.env` for your machine:

| Variable | Description |
|----------|-------------|
| `DATASET_DIR` | Root of the map dataset (contains `images/`, `labels/`) |
| `AUG_DATASET_DIR` | Root of the raw dataset fed into `data_augmentation.py` |
| `PRETRAINED_MODEL` | YOLO checkpoint to start from (e.g. `yolov8n.pt`) |
| `MODEL_SAVE_DIR` | Directory where `best.pt` is copied after training |
| `MODEL_SAVE_NAME` | Filename for the saved model |

### Model weights

Pre-trained YOLO weights are **not committed** to the repository.  
Download `yolov8n.pt` from the [Ultralytics assets releases](https://github.com/ultralytics/assets/releases) and place it in `map_detection/` (or set `PRETRAINED_MODEL` in `.env` to an absolute path).
