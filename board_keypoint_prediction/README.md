# Board Keypoint Prediction

Predicts the 9 hex-center keypoints on the board game board using a YOLOv8-pose model, given a cropped or full image of the board.

## What it does

Each board has 9 hexagonal cell centers that serve as spatial reference points. This module takes an image containing the board and outputs the (x, y) pixel coordinates and visibility flag for each of the 9 keypoints. The model is a fine-tuned YOLOv8m-pose model trained on augmented board images with keypoint labels exported from Label Studio.

## Where it fits in the pipeline

- **Input**: Raw game photo or board-cropped image (JPG/PNG)
- **Requires**: `board_detection/` output (bounding box) to optionally crop the board region first
- **Output**: 9 keypoint coordinates `[(x1,y1), ..., (x9,y9)]` with visibility flags — used to establish the canonical board grid
- **Feeds into**: `homography/` (uses keypoints to compute the perspective transform to the canonical 2D template)

## How to run

### 1. Convert Label Studio annotations

```bash
cd board_keypoint_prediction/src
python convert_to_keypoint.py
```

Reads the Label Studio JSON export (`LABEL_STUDIO_JSON` in `.env`) and writes YOLO pose `.txt` label files to `LABEL_OUTPUT_DIR`.

### 2. Augment training data

```bash
python keypoint_dataaug.py
```

Reads images and YOLO pose labels from `AUG_DATASET_DIR`, applies perspective distortion, rotation, brightness/contrast, and blur, then writes augmented copies to `images_aug/` and `labels_aug/`.

Tune augmentation behaviour directly in `keypoint_dataaug.py`:
- `N_AUGS` — augmented copies per image (default `10`)
- `NUM_KEYPOINTS` — must match number of hex centers (default `9`)

### 3. Train the model

Open and run `src/board_keypoint.ipynb` cell by cell:

| Step | What it does |
|------|-------------|
| 1 | Resolves dataset paths from `.env` |
| 2 | Defines keypoint/class config (9 hex centers, 1 class) |
| 3 | Splits augmented data 70/20/10 into train/val/test |
| 4 | Creates YOLO dataset directory structure |
| 5 | Copies files to splits |
| 6 | Writes `dataset.yaml` for YOLO pose |
| 7 | Visualization helper |
| 8 | Visualises training samples |
| 9 | Trains YOLO pose — tune `EPOCHS`, `IMAGE_SIZE`, `BATCH_SIZE`, `RUN_NAME` here |
| 10 | Runs inference on test samples |
| 11 | Evaluates PCK metrics on test set |


### 4. Run inference with a trained model

```python
from ultralytics import YOLO

model = YOLO("runs/pose/board_pose/weights/best.pt")
results = model("path/to/image.jpg")
keypoints = results[0].keypoints.xy[0]  # shape (9, 2) — pixel (x, y) per keypoint
```

### 5. Utility scripts

| Script | Purpose |
|--------|---------|
| `find_coordinates.py` | Click on hex centers interactively to extract pixel coords (set `IDEAL_IMAGE_PATH` in `.env`) |
| `verify_augmentations.py` | Generate preview images with bounding boxes to check augmentation quality |

## Setup

### Environment

```bash
conda create -n futurescape python=3.9
conda activate futurescape
pip install -r ../requirements.txt
```

### Configure paths

Edit `board_keypoint_prediction/.env` for your machine:

| Variable | Description |
|----------|-------------|
| `KEYPOINT_BASE_DIR` | Path to `board_keypoint_prediction/` (used by notebooks to resolve all sub-paths) |
| `AUG_DATASET_DIR` | Root of dataset for `keypoint_dataaug.py` (contains `images/`, `labels/`) |
| `LABEL_STUDIO_JSON` | Label Studio JSON export file path |
| `LABEL_OUTPUT_DIR` | Output directory for converted YOLO pose labels |
| `VERIFY_DATASET_DIR` | Dataset root for `verify_augmentations.py` |
| `IDEAL_IMAGE_PATH` | Reference image for `find_coordinates.py` |
| `PRETRAINED_MODEL` | YOLO pose checkpoint (e.g. `yolov8m-pose.pt`) |

### Model weights

Pre-trained YOLO pose weights are **not committed** to the repository.  
Download `yolov8m-pose.pt` (or another variant) from the [Ultralytics assets releases](https://github.com/ultralytics/assets/releases) and place it in `board_keypoint_prediction/` (or set `PRETRAINED_MODEL` in `.env` to an absolute path).
